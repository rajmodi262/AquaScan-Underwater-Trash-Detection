---

### **Slide 2: The Problem**
*   **Slide Title:** A Growing Threat to Marine Life
*   **Talking Points:**
    *   **8M+ Tons:** The amount of plastic entering our oceans annually.
    *   **700+ Species:** The number of marine animals directly affected by ocean debris.
    *   **The Technical Hurdle:** Identifying underwater trash is incredibly difficult due to low visibility, light distortion, and visually complex coral environments.
*   **Design Instructions:** Dark navy background. Frosted glass panel on the right containing the text. On the left, use high-contrast imagery of ocean pollution. Use glowing orange/red accent colors for the statistics to convey urgency.

---

### **Slide 3: The Solution**
*   **Slide Title:** Introducing AquaScan AI
*   **Talking Points:**
    *   A scalable software solution leveraging computer vision to detect anthropogenic (human-made) trash in complex aquatic environments.
    *   Algorithm strictly ignores natural features, marine life, and coral using targeted whitelist classification.
    *   Packaged in a world-class, premium web interface for accessibility and ease of use.
*   **Design Instructions:** Split screen layout. Left: Bullet points. Right: A beautifully framed, floating mockup of the AquaScan AI dashboard glowing against a dark water background.

---

### **Slide 4: System Architecture**
*   **Slide Title:** High-Level Architecture
*   **Talking Points:**
    *   **Frontend UI:** Built with React, Framer Motion, and Tailwind CSS for a premium cinematic feel.
    *   **Backend API:** Python-based processing pipeline.
    *   **Core Engine:** OpenCV utilizing multi-pass grid analysis and adaptive scoring.
*   **Design Instructions:** A sleek, dark-mode flowchart. Use glowing neon cyan lines to show the data flow: *Image Upload → Preprocessing → Grid Analysis → Heatmap Generation → Client UI*.

---

### **Slide 5: Core Computer Vision Tech**
*   **Slide Title:** Precision Detection Pipeline
*   **Talking Points:**
    *   **Color & Edge Filtering:** Strips away ambient blue/green water hues to isolate unusual shapes and sharp edges.
    *   **Grid-Based Scanning:** Divides high-resolution images into small micro-cells for hyper-localized analysis.
    *   **Adaptive Z-Score Scoring:** Calculates anomalies statistically rather than using rigid thresholds, practically eliminating false positives from fish.
*   **Design Instructions:** 3-column layout inside glassmorphism cards. Show a visual progression: *Raw Image → Grid Overlay → Final Heatmap Score*.

---

### **Slide 6: Key Platform Features**
*   **Slide Title:** Turning Data into Insights
*   **Talking Points:**
    *   **Smart Upload:** Drag-and-drop processing for JPEGs, PNGs, and WebPs.
    *   **Visual Insights:** Dynamic heatmaps overlaying the original image to pinpoint high-probability debris zones.
    *   **Interactive Tools:** Comparison sliders allow users to see exactly how the algorithm processed the image "before and after".
*   **Design Instructions:** Feature a high-res screenshot or animated GIF of the comparison slider in action, framed inside a sleek laptop mockup.

---

### **Slide 7: User Interface & UX**
*   **Slide Title:** Premium Cinematic Experience
*   **Talking Points:**
    *   Designed to feel like an investor-ready, high-end SaaS product rather than a basic academic prototype.
    *   Features real underwater video backgrounds and responsive parallax scrolling.
    *   Utilizes "Glassmorphism 2.0"—frosted, translucent panels that maintain ocean context while highlighting critical data.
*   **Design Instructions:** A collage of UI elements (the animated counters, the cursor light effect, the glass cards). Dominant colors should be deep navy, bright cyan, and translucent white.

---

### **Slide 8: Performance Metrics**
*   **Slide Title:** Precision and Reliability
*   **Talking Points:**
    *   Achieved high precision by transitioning from heuristic blacklists to a strict anthropogenic whitelist.
    *   Significantly reduced false positives on living marine species compared to earlier iterations.
    *   Highly optimized OpenCV pipeline capable of processing high-resolution imagery without memory bottlenecks.
*   **Design Instructions:** Use glowing circular progress rings or animated bar charts showing precision and processing speed. Use green/teal gradients for positive metrics to emphasize success.

---

### **Slide 9: Technical Challenges**
*   **Slide Title:** Overcoming the Deep
*   **Talking Points:**
    *   **Challenge:** Color distortion and light scattering underwater.
        *   *Solution:* Implemented adaptive equalization and contrast stretching in preprocessing.
    *   **Challenge:** Misclassifying silver fish as plastic debris.
        *   *Solution:* Enforced strict geometric filters and shifted to standard deviation (Z-score) anomaly detection.
*   **Design Instructions:** Side-by-side comparison layout. Show a challenging image with poor lighting next to an image where the algorithm successfully isolated a plastic bottle despite the conditions.

---

    *   Transitioning from static image analysis to real-time video feed processing.

*   **Design Instructions:** A visionary, conceptual image of an underwater drone emitting a scanning light, overlaid with subtle, techy UI bounding boxes.
